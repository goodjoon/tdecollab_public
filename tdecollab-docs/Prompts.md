
사내 confluence, jira, gitlab on-premise 시스템들과 interaction 하면서 PRD 를 정의하고
   기능정의서, 테스트설계서, UI/UX 정의서 등을 confluence 로 만들고, jira 에 개발 task 들을 ticket 으로 만들어서
   올리고, gitlab 을 통해 개발 코드가 push 되면 관련 jira ticket 이 업데이트 되는 등의 자동화 플랫폼과 agentic mode 의
   ai 개발도구를 사용해 개발을 시켜 위 흐름을 따르며 개발하도록 하는 harness 도 만들려고 한다.

TDE Collab 도구는 사내의 On-Premise 타입의 Gitlab, Jira, Confluence 들을 활용하여 PRD 를 생성하면 PRD 를 더 명확하게 작성 할 수 있게 사용자와 interaction 하고, 기능정의서를 작성하고, 단위 테스트 계획서, 통합 테스트 계획서를 작성하며, 개발 Task 들을 정의하고 이러한 문서들은 사내 confluence 에 업로드 하며, 각 개발 Task 들을 JIRA ticket 으로 만들어서 사내 jira 에 ticket 을 업로드 해준다.
